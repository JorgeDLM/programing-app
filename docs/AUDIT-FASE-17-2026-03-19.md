## 4. Cost Tracking

### Servicio creado: `src/lib/shared/cost-tracker.ts`

**Pricing implementado (USD per million tokens):**
| Modelo | Input/MTok | Output/MTok |
|---|---|---|
| Claude Opus 4.6 | $15.00 | $75.00 |
| Claude Sonnet 4.6 | $3.00 | $15.00 |
| Claude Haiku 4.5 | $0.80 | $4.00 |

**Funciones:**
- `estimateCost(model, tokensInput, tokensOutput)` → CostEstimate con desglose
- `estimatePipelineCost(phases[])` → costo estimado del pipeline completo
- `formatCost(cost)` → formato legible ($X.XXXX o $X.XX¢)

**Integrado en:**
- Inspector API (`/api/inspect/[id]`) — agrega `costEstimate` al response de tickets
- InspectorPanel UI — muestra costo estimado, modelo, tokens in/out en seccion Telemetry & Cost

**Nota:** Los costos son estimados basados en tokensUsed (que combina input+output). Se usa un split 60/40 aproximado cuando no hay datos separados. Los valores son etiquetados como `isEstimated: true`.

## 6. Heuristicas Auditables

### Templates pipeline
| Regla | Resultado |
|---|---|
| Ticket con keywords "api, endpoint, prisma, database..." | backend-heavy → usa desarrollador-backend en vez de implementador |
| Ticket con keywords "ui, componente, formulario, tailwind..." | frontend-heavy → usa desarrollador-frontend |
| Sin signal clara de backend/frontend | usa implementador generico |
| ticketType architecture_change | usa evaluador-de-riesgos en discovery |
| ticketType quotation | usa estratega-de-producto + evaluador-de-riesgos |
| Todas las features | guardian-de-seguridad en review + documentador en finalization |

### Consultation teams
| Regla | Resultado |
|---|---|
| Session type tiene componente de riesgo | incluye evaluador-de-riesgos |
| Session type tiene componente de UX | incluye disenador-de-experiencia |
| Session type es growth/marketing | incluye mercadologo, especialista-en-conversion, experto-en-ventas |
| Session type es technical | incluye disenador-de-base-de-datos |
| Session type es quotation | incluye estratega-de-producto |
