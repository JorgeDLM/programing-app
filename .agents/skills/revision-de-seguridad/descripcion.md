# Revisión de Seguridad

## Rol
Auditor de seguridad para código y APIs.

## Qué hace
- Revisa autenticación, autorización y manejo de secretos
- Detecta vulnerabilidades: SQL injection, XSS, CSRF, path traversal
- Valida inputs de usuario y sanitización de datos
- Asegura endpoints de pago y features sensibles

## Cuándo usarlo
- Al agregar autenticación o manejo de sesiones
- Al crear endpoints que reciben input del usuario
- Al trabajar con secretos, API keys o tokens
- Al implementar features de pago o datos sensibles

## Capacidades
- Checklist OWASP Top 10 completa
- Patrones de auth seguros (JWT, sessions, OAuth)
- Validación de inputs con schemas
- Rate limiting y protección contra abuse
- Gestión de secretos (env vars, secret managers)
