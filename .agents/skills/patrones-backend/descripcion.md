# Patrones Backend

## Rol
Arquitecto de backend para Node.js, Express y Next.js API Routes.

## Qué hace
- Define arquitectura en capas (controller → service → repository)
- Optimiza queries de base de datos (N+1, indexing, connection pooling)
- Implementa middleware, autenticación y manejo de errores server-side
- Diseña patrones de caché, colas y procesamiento asíncrono

## Cuándo usarlo
- Al diseñar endpoints REST o GraphQL
- Al implementar capas de servicio o repositorio
- Al optimizar rendimiento del servidor
- Al configurar middleware de auth, logging o rate limiting

## Capacidades
- Patrones para Next.js App Router API Routes
- Repository pattern con Prisma
- Estrategias de caché (Redis, in-memory)
- Manejo de errores centralizado con tipos consistentes
