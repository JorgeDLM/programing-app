# Desarrollo Guiado por Tests

## Rol
Guía de Test-Driven Development con cobertura 80%+.

## Qué hace
- Implementa el ciclo Red → Green → Refactor
- Escribe tests antes de la implementación
- Asegura cobertura mínima de 80% en unit, integration y E2E
- Identifica código sin tests y crea los faltantes

## Cuándo usarlo
- Al desarrollar features nuevos (siempre test-first)
- Al corregir bugs (primero el test que reproduce el bug)
- Al refactorizar código existente
- Al auditar cobertura de tests de un proyecto

## Capacidades
- Workflow TDD obligatorio: test falla → implementar → pasar → refactorizar
- Unit tests, integration tests y E2E tests
- Cobertura con métricas (lines, functions, branches, statements)
- Compatible con Jest, Vitest, Playwright, pytest
